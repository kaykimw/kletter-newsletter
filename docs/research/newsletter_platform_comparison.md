# 뉴스레터 플랫폼 리서치 (2026.04.16)

## 조사 배경
- WATCHA 일본사업팀 Weekly OTT 트렌드 리포트 발행
- 사내 공유용 → 추후 일본 유저 배포용 확장 예정
- 매주 1회 정기 발행, 자동화 필요
- 커스텀 HTML 디자인 사용
- 웹페이지도 함께 운영 예정

## 1. 뉴스레터 플랫폼 (SaaS)

### 1-1. 스티비 (Stibee)
- **사이트:** https://stibee.com
- **무료 한도:** 구독자 500명, 월 10,000통
- **API 자동발송:** 유료만 (스탠다드 월 16,500원~)
- **HTML 커스텀:** O (HTML 직접 입력 지원)
- **한국어 UI:** O
- **장점:** 한국어 완벽 지원, 드래그앤드롭 에디터, 메일머지 무료
- **단점:** 무료에서 API 발송 불가, 이메일 클라이언트에서 CSS 변수(var) 미지원으로 커스텀 HTML 깨짐 가능
- **적합도:** 수동 발송 + 한국어 UI 원할 때

### 1-2. Mailchimp
- **사이트:** https://mailchimp.com
- **무료 한도:** 구독자 500명, 월 1,000통
- **API 자동발송:** 무료 가능
- **HTML 커스텀:** O
- **한국어 UI:** X (영어)
- **장점:** 글로벌 표준, 무료 API 지원, 마케팅 자동화 기본 제공
- **단점:** 무료 한도가 매우 적음 (월 1,000통), 영어 UI
- **적합도:** 소규모 시작 가능하나 확장성 부족

### 1-3. Brevo (구 Sendinblue)
- **사이트:** https://brevo.com
- **무료 한도:** 하루 300통 (월 9,000통), 구독자 무제한
- **API 자동발송:** 무료 가능
- **HTML 커스텀:** 완전 자유
- **한국어 UI:** X (영어)
- **열람률/클릭률 분석:** 무료 제공
- **장점:** 무료 한도 가장 넉넉, 무료에서 분석 가능, 구독자 세그먼트/A/B 테스트 지원
- **단점:** 영어 UI, API가 Resend보다 복잡
- **적합도:** 무료에서 최대한 많은 기능을 쓰고 싶을 때 최적

### 1-4. Resend
- **사이트:** https://resend.com
- **무료 한도:** 트랜잭셔널 월 3,000통 (하루 100통) / 마케팅 1,000명 구독자 무제한
- **API 자동발송:** 무료 가능
- **HTML 커스텀:** 완전 자유 + React Email 지원
- **한국어 UI:** X (영어)
- **열람률/클릭률 분석:** 유료만 ($20/월~)
- **장점:** API가 업계 최고 수준으로 간결 (코드 3줄), React Email로 웹+이메일 통합 가능, Audiences(구독자 관리) 내장
- **단점:** 무료에서 분석 불가, 하루 100통 제한
- **유료 플랜:** Pro $20/월 (50,000통), Scale $90/월 (100,000통)
- **적합도:** 웹페이지와 이메일을 하나의 프로젝트로 관리하고 싶을 때 최적

### 1-5. Beehiiv
- **사이트:** https://beehiiv.com
- **무료 한도:** 구독자 2,500명
- **API 자동발송:** 유료만 ($31.20/월~)
- **HTML 커스텀:** 제한적
- **장점:** 뉴스레터 특화, 랜딩 페이지 제작 가능, 12+ 직접 연동
- **단점:** 무료에서 API 불가, HTML 자유도 낮음
- **적합도:** 뉴스레터 전용 브랜드 페이지가 필요할 때

### 1-6. Substack
- **사이트:** https://substack.com
- **무료 한도:** 구독자 무제한
- **API 자동발송:** X
- **HTML 커스텀:** X (자체 포맷만)
- **장점:** 완전 무료, 구독자 무제한, 유료 구독 멤버십 기능
- **단점:** HTML 커스텀 불가 → 지금 만든 디자인 사용 불가, API 없음
- **적합도:** 커스텀 디자인 불필요하고 텍스트 중심 뉴스레터일 때

### 1-7. 메일리 (Maily)
- **사이트:** https://maily.so
- **무료 한도:** 개인 무료 무제한
- **API 자동발송:** X
- **HTML 커스텀:** 제한적
- **한국어 UI:** O
- **장점:** 개인 무료, 노션 같은 깔끔한 UI, 유료 구독 멤버십 가능
- **단점:** API 없음, HTML 자유도 낮음
- **적합도:** 개인 뉴스레터, 텍스트 중심

## 2. 직접 발송 방식

### 2-1. Gmail SMTP
- **비용:** 완전 무료
- **한도:** 하루 500통 (개인 Gmail) / 2,000통 (Google Workspace)
- **필요한 것:** Gmail 2단계 인증 + 앱 비밀번호 설정
- **장점:** 추가 서비스 가입 없음, 기존 계정 사용, 수신자에게 익숙한 발신자
- **단점:** 구독자 관리 없음, 분석 없음, 대량 발송 시 스팸 위험
- **적합도:** 사내 소규모 (10~50명) 즉시 시작

### 2-2. Google Apps Script + 스프레드시트
- **비용:** 완전 무료
- **한도:** 하루 100통 (무료 Gmail) / 1,500통 (Workspace)
- **필요한 것:** 구글 시트 + Apps Script 코드
- **장점:** 구글 계정만 있으면 가능, 시간 예약 발송, 스프레드시트로 수신자 관리
- **단점:** 디자인 제약, 대량 발송 불가
- **적합도:** 사내 10~20명 소규모

### 2-3. Amazon SES
- **사이트:** https://aws.amazon.com/ses/
- **비용:** 월 3,000통 무료 (EC2에서 발송 시), 이후 1,000통당 ~$0.10
- **필요한 것:** AWS 계정 + SES 설정 + 도메인 인증 (SPF/DKIM)
- **장점:** 가장 저렴한 대량 발송, 안정적 인프라, 무제한 확장
- **단점:** 초기 설정 복잡, 샌드박스 해제 필요, 개발 지식 필요
- **적합도:** 유저 수천~수만 명 대규모 확장 시

### 2-4. Nodemailer (Node.js 라이브러리)
- **사이트:** https://nodemailer.com
- **비용:** 무료 (SMTP 서비스는 별도)
- **필요한 것:** Node.js + SMTP 서비스 (Gmail/SES/Brevo 등)
- **장점:** 완전한 코드 제어, HTML/첨부파일/임베드 이미지 지원
- **단점:** SMTP를 별도로 준비해야 함, 개발 필요
- **적합도:** 자동화 스크립트에서 이메일 발송할 때 (다른 SMTP와 조합)

## 3. 자체 호스팅 (오픈소스)

### 3-1. Listmonk
- **사이트:** https://listmonk.app
- **GitHub:** https://github.com/knadh/listmonk (v6.0.0, 2026.01)
- **비용:** 소프트웨어 무료, 서버 비용만 (VPS 월 $5~)
- **필요한 것:** 서버, Docker, SMTP 서비스 연결
- **장점:** 구독자/발송량 무제한, 대시보드에서 캠페인 관리, 열람률/클릭률 분석, 멀티 SMTP 큐
- **단점:** 서버 관리 필요, 초기 세팅 필요
- **적합도:** 장기적 비용 절감 + 완전한 통제 원할 때

### 3-2. Keila
- **사이트:** https://www.keila.io
- **비용:** 소프트웨어 무료 (오픈소스)
- **필요한 것:** Docker, SMTP/AWS SES/Mailgun 등 연결
- **장점:** Mailchimp 대안, 100% 오픈소스, 프리미엄 기능 없음 (전부 무료)
- **단점:** Listmonk 대비 커뮤니티 작음
- **적합도:** Listmonk 대안

### 3-3. SendPortal
- **사이트:** https://sendportal.io
- **비용:** 무료 (오픈소스)
- **장점:** 무제한 유저/구독자/세그먼트/메시지
- **단점:** 업데이트 빈도 낮음

## 4. 웹페이지 호스팅 옵션

### 4-1. Vercel
- **비용:** 무료 (Hobby 플랜)
- **장점:** GitHub 연동 자동 배포, 빠른 CDN, Next.js 최적화
- **적합도:** 웹페이지 + Resend 연동 시 최적

### 4-2. Netlify
- **비용:** 무료 (Starter 플랜)
- **장점:** 드래그앤드롭 배포 가능, 폼 기능 내장
- **적합도:** 단순 HTML 호스팅

### 4-3. GitHub Pages
- **비용:** 무료
- **장점:** GitHub 레포 연동, 설정 간단
- **단점:** 정적 파일만 가능 (서버 기능 X)
- **적합도:** HTML 파일만 공개할 때

### 4-4. Notion
- **비용:** 무료
- **장점:** 세팅 불필요, 링크 공유만으로 완료
- **단점:** HTML 커스텀 불가, 외부 공개 시 브랜딩 제한
- **적합도:** 사내 공유 임시 방편

## 6. 최종 추천 (3가지)

### 추천 1: Resend + Vercel (최적)
- 웹페이지와 이메일을 하나의 프로젝트에서 관리
- API 가장 간결, 자동화 안정적
- 마케팅 이메일 1,000명까지 무제한 무료
- 유저 배포 확장에 가장 자연스러움

### 추천 2: Brevo (분석 중시)
- 무료에서 열람률/클릭률 분석 가능 (Resend는 유료)
- 월 9,000통으로 발송 여유 가장 큼
- 웹페이지는 별도 호스팅 필요

### 추천 3: Gmail SMTP + Vercel (즉시 시작)
- 추가 서비스 가입 없이 바로 시작
- 사내 소규모 공유에 충분
- 유저 배포 단계에서 Resend/Brevo로 전환 필요
