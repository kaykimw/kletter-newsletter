## 구독 시스템 상세

### 구독 흐름
1. 유저가 `/subscribe.html` 에서 이메일 입력 → "구독하기" 클릭
2. 클라이언트에서 이메일 정규식 검증 (`/^[^\s@]+@[^\s@]+\.[^\s@]+$/`)
3. `POST /api/subscribe` 호출 (JSON body: `{email}`)
4. 서버에서:
   - CORS 헤더 설정
   - Rate limiting 체크 (같은 IP에서 10초 내 재요청 차단)
   - 이메일 정규식 재검증
   - Resend API 호출 → Audience에 contact 추가
5. 성공 시 "구독 완료! 매주 금요일 뉴스레터를 보내드릴게요." 메시지

### Rate Limiting 주의사항
`subscribe.js`의 `recentRequests` Map은 서버리스 함수의 메모리에 저장되므로, Vercel에서 함수가 새 인스턴스로 뜨면 초기화됨. 완벽한 rate limiting은 아니지만 기본적인 보호는 됨.
